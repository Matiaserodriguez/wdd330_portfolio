Supplying request options

The fetch() method can be used in an asynchronous function with the keyword "await" before it, or using ".then()" method.
fetch() can optionally accept a second parameter where we can specify the HTTP requests methods such as GET, POST, PUT, DELETE, as well as the mode no-cors, *cors, same-origin (used to get permission to access selected resources from a server, on a different origin (domain) to which it belongs. Inside of this same parameter we can specify: "cache", "credentials", "redirect", "referrerPolicy" and both the "headers" and the "body" in case it's necessary (for instance, we don't pass anything to the body in a GET request).
